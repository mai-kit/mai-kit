# @mai-kit/draw

将舞萌 DX 成绩数据渲染为海报 / Best 板 / 单曲卡 / 升分板。入口扁平：
`new Draw({ database })` 后 **一种图一个方法**（成对提供 PNG / SVG），按图传入最小数据。
数据通过结构化接口注入，draw 不绑定具体查分服务或游戏数据库。

## 安装

```bash
pnpm add @mai-kit/draw
```

## 使用

```ts
import { writeFileSync } from "node:fs";
import { Draw } from "@mai-kit/draw";

// profile / bests：任意 prober 适配
// database：满足素材 / 标签等能力的数据源（如任意 MaimaiDatabase）
const draw = new Draw({ database });

const poster: Uint8Array = await draw.poster(profile, bests);
writeFileSync("poster.png", poster);
const posterSvg: string = await draw.posterSvg(profile, bests);
```

### `RenderOptions`（各方法末位可选）

所有出图方法最后一参都是可选的 `RenderOptions`：

| 字段            | 类型                         | 默认                       | 说明                                            |
| --------------- | ---------------------------- | -------------------------- | ----------------------------------------------- |
| `scale`         | `number`                     | `2`                        | 相对逻辑画布 1920×1080 的倍率；`2` → 3840×2160  |
| `fonts`         | satori 字体表                | `@mai-kit/assets` 默认字体 | 整表覆盖 Noto Sans SC + Comfortaa               |
| `footerLeft`    | `string`                     | 无                         | 画布左下页脚；与 `footerRight` 都不传则不画页脚 |
| `footerRight`   | `string`                     | 无                         | 画布右下页脚                                    |
| `assetFallback` | `"error"` \| `"placeholder"` | `"error"`                  | 封面/头像失败：抛错，或用内置占位图继续         |

```ts
const opts = {
  scale: 1,
  footerLeft: "my-app",
  footerRight: "generated by mai-kit",
  assetFallback: "placeholder" as const,
};

await draw.poster(profile, bests, opts);
await draw.best15({ name: profile.name, rating: profile.rating }, bests, opts);
await draw.chart(scoreChart, opts);
```

尺寸查询：`Draw.getPosterSize(scale)` / `getBestSize(scale)` / `getBoardSize(scale)`。

### 出图方法（一种图一个）

| 方法                       | 位置参数                                      | 末位 options    | 用途                    |
| -------------------------- | --------------------------------------------- | --------------- | ----------------------- |
| `poster` / `posterSvg`     | `profile` + `bests`，或已有 `PosterData`      | `RenderOptions` | 完整 B50 海报（含雷达） |
| `best15` / `best15Svg`     | 署名 + `Bests`（内部取 `dx` 前最多 15）       | `RenderOptions` | 新曲 B15；只切不排      |
| `best35` / `best35Svg`     | 署名 + `Bests`（内部取 `standard` 前最多 35） | `RenderOptions` | 旧曲 B35；只切不排      |
| `best50` / `best50Svg`     | 署名 + `Bests`（内部 15+35 拼接）             | `RenderOptions` | B50 全曲；只切不排      |
| `chart` / `chartSvg`       | 单曲 `ScoreChart`                             | `RenderOptions` | 单曲成绩卡（无玩家栏）  |
| `upgrades` / `upgradesSvg` | `UpgradeBoardData`                            | `RenderOptions` | 加分推荐板              |

#### Best 板

传 prober 的 `Bests` 即可；库内按源顺序切割（**不重排**）。数量不足则有几画几，不补假数据：

```ts
const player = { name: profile.name, rating: profile.rating };
await draw.best15(player, bests); // dx 前最多 15
await draw.best35(player, bests); // standard 前最多 35
await draw.best50(player, bests); // 新 15 + 旧 35
```

#### 单曲卡

只展示谱面成绩，不画玩家署名：

```ts
const png = await draw.chart(scoreChart, { scale: 1 });
```

#### 加分板

draw **不**依赖 analysis；候选由宿主计算。
正确语义：**全曲成绩里，刷到目标后能抬 B15/B35 总分**的谱面（不是随便抬单曲达成率）。

```ts
import { rankBestsUpgradeCandidates } from "@mai-kit/analysis";
import { buildSongLevelMap, scoreMapKey } from "@mai-kit/utils/song";

const scores = await player.getScores();
const { songs } = await database.getSongList();
const levelMap = buildSongLevelMap(songs);
const songVersion = new Map<number, number>();
for (const song of songs) {
  if (song.version !== undefined) songVersion.set(song.id, song.version);
}
const versions = [...songVersion.values()];
if (versions.length === 0) throw new Error("database does not provide numeric versions");
const currentVersion = Math.max(...versions);
const entries = scores.flatMap((score) => {
  const levelValue = levelMap.get(scoreMapKey(score));
  return levelValue == null ? [] : [{ score, levelValue }];
});
const bests = await player.getBests();
const candidates = rankBestsUpgradeCandidates(entries, {
  currentBests: bests,
  isNewSong: (s) => songVersion.get(s.id) === currentVersion,
  minRate: "sssp", // 目标至少 SSS+；不限制评级时改传 targetAchievement
  limit: 10,
});

const png = await draw.upgrades({
  candidates,
});
```

### 海报数据聚合

`draw.poster(profile, bests)` 内部会聚合标签与（可选）曲目物量。需要先看
`PosterData` 时可用包根 `buildPosterData`：

```ts
import { buildPosterData, Draw } from "@mai-kit/draw";

const data = await buildPosterData(profile, bests, database);
const png = await new Draw({ database }).poster(data);
```

数据源带曲目列表时，会自动补全 DX 满分 / 定数等展示字段。
每次出图只读取一次曲目列表并建立一次索引；Best50 不会按 50 张卡重复扫描完整曲目表。

### 素材来源（database）

`Draw` 通过构造配置 `DrawOptions.database` 注入数据 / 素材层。渲染前按
`charts[].id`（封面）/ `player.icon.id`（头像）预拉素材并转 data URI。
昵称旁展示 `@mai-kit/assets` 的段位（`course_rank`）与阶级（`class_rank`）徽章，不拉姓名框。
类型上只需结构化满足最小 `AssetSource`、`ChartTagSource`（海报）与可选 `SongListSource`，
**不硬依赖** `@mai-kit/database`：

```ts
import { LxnsMaimaiDatabase } from "@mai-kit/database";
import { Draw } from "@mai-kit/draw";

const database = new LxnsMaimaiDatabase();
const draw = new Draw({ database });
```

若 database 在创建时配置了 `DatabaseCacheStore`，draw 会自然复用其缓存；draw 本身不
切换或重复实现跨玩家缓存。多曲封面按输入顺序返回，内部最多同时加载 8 项，避免浏览器
或 CDN 被 B50 的瞬时并发压满。

`assetFallback` 详见上文 [RenderOptions](#renderoptions各方法末位可选)。
已提供 `coverDataUri` / `avatarDataUri` 时不会再走 database 拉取。

## 数据结构

`PosterData` 只用于完整海报：`player` / `charts` / `summary` / `radar`（可选
`personalMetrics` 整表覆盖网格）。评级分布、定数直方图等由 `charts` 在渲染时派生。
成绩码值采用 prober 原始字段，格式化在渲染时完成。

```ts
interface ScoreChart {
  id: number; // 曲目 id，Draw 按 id 拉封面
  song_name?: string;
  type: SongType; // 标签关联
  level_index: LevelIndex; // 标签关联
  achievements: number; // 原始达成率，渲染时格式化为 "100.8621%"
  dx_score: number;
  dx_max?: number; // 满分（notes.total×3），有值时显示为 "dx_score/dx_max"
  dx_star?: number; // 0–5，与 dx_score 同排画星（1–2 / 3–4 / 5 档不同素材）
  level?: string;
  rate?: RateType; // 渲染时映射为 "SSS+" 等
  fc?: FCType | null; // 渲染时派生徽章（AP+/AP/FC/...）
  fs?: FSType | null; // 渲染时派生徽章（SYNC/FS/...）
  coverDataUri?: string; // 预解析封面（覆盖 id 拉取）
  coverPath?: string;
  badgeImageUri?: string;
}
```

```ts
interface PlayerProfile {
  name: string;
  rating: number;
  course_rank?: number; // 昵称旁段位徽章（含 0=初学者）
  class_rank?: number; // 昵称旁阶级徽章（0–25）
  icon?: { id?: number }; // 有真实 id 时 Draw 才拉头像
  upload_time?: string;
  avatarDataUri?: string;
  avatarPath?: string;
}
```

完整海报要求 database 满足 `ChartTagSource`。若 B50 无法匹配至少三个配置标签会明确抛出
`DrawError`，不会发明占位雷达。Best 板 / 单曲卡不拉标签。

完整字段见 `src/types.ts`（`PosterData` / `PlayerProfile` / `PosterSummary` /
`RadarItem` / `MetricItem` 等）。

## Node 与浏览器

| API                  | Node                    | Web                                               |
| -------------------- | ----------------------- | ------------------------------------------------- |
| `poster` 等 → PNG    | ✅ resvg-js             | ✅ resvg-wasm                                     |
| `*Svg` → SVG         | ✅                      | ✅                                                |
| 落盘 / 下载          | 宿主 `fs` 等            | 宿主 `Blob` 等                                    |
| 默认字体 / wasm      | ✅ 经 `@mai-kit/assets` | ✅ 同上（bundler 保留 assets 包内 fonts / resvg） |
| 本地磁盘 `coverPath` | ✅                      | ❌ 用 data/http(s)/blob URI                       |

## 开发与测试

```bash
pnpm --filter @mai-kit/draw test        # 单测（固定数据，先 build）
pnpm --filter @mai-kit/draw test:integration:lxns   # 真数据集成（prober + database + 渲染）
pnpm test:web                           # Vite 构建 + headless Chrome 双端 smoke
```

`test:integration:lxns` 读取仓库根 `.env` 的 `LXNS_API_PERSONAL_ACCESS_TOKEN`；未设置则 skip。
产物写在 `packages/draw/output/`（已 gitignore）。

## 错误

字体加载、SVG 渲染或栅格化失败时抛出 `DrawError`（继承自 `@mai-kit/shared` 的
`MaiKitError`），可用 `isDrawError` 或 `isMaiKitError` 捕获。
