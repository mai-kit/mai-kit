# @mai-kit/draw

将舞萌 DX 玩家数据渲染为成绩海报或 Best 图。`Draw.withPlayer(profile, bests)` 负责聚合
玩家数据、曲目和素材，并返回可渲染多个版式的 `PlayerDraw`。数据通过结构化接口注入，
draw 不绑定具体查分服务或游戏数据库。

## 安装

```bash
pnpm add @mai-kit/draw
```

## 使用

```ts
import { writeFileSync } from "node:fs";
import { Draw } from "@mai-kit/draw";

// profile / bests：任意 prober 适配
// database：满足素材 / 标签等能力的数据源（如任意 MaimaiDatabase）
const playerDraw = await new Draw({ database }).withPlayer(profile, bests);

const poster: Uint8Array = await playerDraw.render("poster");
writeFileSync("poster.png", poster);
const posterSvg: string = await playerDraw.renderSvg("poster");

const best50 = await playerDraw.render("best50");
const best15 = await playerDraw.render("best15");
const best35 = await playerDraw.render("best35");
```

页脚左右可选：

```ts
const png = await playerDraw.render("poster", {
  footerLeft: "my-app",
  footerRight: "generated by mai-kit",
});
```

### 版式（`DrawLayout`）

| `layout` | 内容                                |
| -------- | ----------------------------------- |
| `poster` | 横版完整 B50（左栏档案 + 右栏曲目） |
| `best50` | B50 全曲板（5 列 × 10 行）          |
| `best15` | 新曲 B15 板（3 列 × 5 行）          |
| `best35` | 旧曲 B35 板（5 列 × 7 行）          |

入口：`playerDraw.render(layout, options?)`（PNG）/ `playerDraw.renderSvg(layout, options?)`（SVG）。  
尺寸：`Draw.getSize(layout, scale)`，或 `getPosterSize` / `getBestSize`（均为 1920×1080 × scale）。

### 绑定玩家数据

`Draw.withPlayer()` 把玩家档案 + Best50 与标签（及可选曲目物量）聚合一次，返回持有
`PosterData` 的 `PlayerDraw`。数据源带曲目列表时，会自动补全 DX 满分 / 定数等展示字段：

```ts
const playerDraw = await new Draw({ database }).withPlayer(profile, bests);
const png = await playerDraw.render("poster", { scale: 2 });
```

聚合结果可从 `playerDraw.data` 读取。已有完整 `PosterData` 时可直接：

```ts
import { PlayerDraw } from "@mai-kit/draw";

const playerDraw = new PlayerDraw(data);
const png = await playerDraw.render("poster");
```

### 素材来源（database）

`Draw` 通过构造配置 `DrawOptions.database` 注入数据 / 素材层。`PlayerDraw` 渲染前按
`charts[].id`（封面）/ `player.icon.id`（头像）预拉素材并转 data URI。
昵称旁展示 `@mai-kit/assets` 的段位（`course_rank`）与阶级（`class_rank`）徽章，不拉姓名框。
类型上只需结构化满足最小 `AssetSource`、`ChartTagSource` 与可选 `SongListSource`，
**不硬依赖** `@mai-kit/database`：

```ts
import { LxnsMaimaiDatabase } from "@mai-kit/database";
import { Draw } from "@mai-kit/draw";

const database = new LxnsMaimaiDatabase();
const draw = new Draw({ database });
```

若 database 在创建时配置了 `DatabaseCacheStore`，draw 会自然复用其缓存；draw 本身不
切换或重复实现跨玩家缓存。

单项素材拉取失败时回退到占位图 / 文字，不影响整体渲染。

## 数据结构

`PosterData` 采用 prober 原始字段（`id` / `song_name` / `achievements:number` /
`dx_score` / `dx_star` / `rate` / `fc` / `fs` / `course_rank` / `class_rank` / `icon` /
`upload_time`），格式化在渲染时完成；prober 的 `Score` / `PlayerProfile` 可直接透传。

```ts
interface ScoreChart {
  id: number; // 曲目 id，Draw 按 id 拉封面
  song_name?: string;
  type: SongType; // 标签关联
  level_index: LevelIndex; // 标签关联
  achievements: number; // 原始达成率，渲染时格式化为 "100.8621%"
  dx_score: number;
  dx_max?: number; // 满分（notes.total×3），有值时显示为 "dx_score/dx_max"
  dx_star?: number; // 0–5，与 dx_score 同排画星（1–2 / 3–4 / 5 档不同素材）
  level?: string;
  rate?: RateType; // 渲染时映射为 "SSS+" 等
  fc?: FCType | null; // 渲染时派生徽章（AP+/AP/FC/...）
  fs?: FSType | null; // 渲染时派生徽章（SYNC/FS/...）
  coverDataUri?: string; // 预解析封面（覆盖 id 拉取）
  coverPath?: string;
  badgeImageUri?: string;
}
```

```ts
interface PlayerProfile {
  name: string;
  rating: number;
  course_rank?: number; // 昵称旁段位徽章（含 0=初学者）
  class_rank?: number; // 昵称旁阶级徽章（0–25）
  icon?: Collection; // Draw 按 icon.id 拉头像
  upload_time?: string;
  avatarDataUri?: string;
  avatarPath?: string;
}
```

`Draw.withPlayer()` 是异步方法，并要求 database 满足 `ChartTagSource`。database 包的
`MaimaiDatabase` 已提供该能力；若 B50 无法匹配至少三个配置标签会明确抛出
`DrawError`，不会创建带占位雷达的玩家对象。

完整字段见 `src/types.ts`（`PosterData` / `PlayerProfile` / `PosterSummary` /
`RatingDistributionItem` / `RadarItem` / `MetricItem`）。

## Node 与浏览器

| API                       | Node                    | Web                                               |
| ------------------------- | ----------------------- | ------------------------------------------------- |
| `render(layout)` → PNG    | ✅ resvg-js             | ✅ resvg-wasm                                     |
| `renderSvg(layout)` → SVG | ✅                      | ✅                                                |
| 落盘 / 下载               | 宿主 `fs` 等            | 宿主 `Blob` 等                                    |
| 默认字体 / wasm           | ✅ 经 `@mai-kit/assets` | ✅ 同上（bundler 保留 assets 包内 fonts / resvg） |
| 本地磁盘 `coverPath`      | ✅                      | ❌ 用 data/http(s)/blob URI                       |

## 字体与静态资源

draw **不自带** `assets/`。默认字体与 Web 用 resvg wasm 都在 `@mai-kit/assets`：

- **Noto Sans SC**（简体 / CJK）、**Comfortaa**（Latin 装饰）
- CSS：`fontFamily: "Noto Sans SC, Comfortaa"`（satori 按字回退）
- `assets/resvg/index_bg.wasm`（仅 Web 栅格化）

自定义字体：`RenderOptions.fonts`（需与 `fontFamily` 名称对齐）。

默认 `scale` 为 `2`：海报与 Best 单图均为 3840×2160（1920×1080 × 2）；需要更小预览图时可显式传入 `{ scale: 1 }`。

## 开发与测试

```bash
pnpm --filter @mai-kit/draw test        # 单测（固定数据，先 build）
pnpm --filter @mai-kit/draw test:integration:lxns   # 真数据集成（prober + database + 渲染）
```

`test:integration:lxns` 读取仓库根 `.env` 的 `LXNS_API_PERSONAL_ACCESS_TOKEN`；未设置则 skip。
产物写在 `packages/draw/output/`（已 gitignore）。

## 错误

字体加载、SVG 渲染或栅格化失败时抛出 `DrawError`（继承自 `@mai-kit/shared` 的
`MaiKitError`），可用 `isDrawError` 或 `isMaiKitError` 捕获。
